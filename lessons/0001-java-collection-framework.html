第一部分：Java集合框架概览

1.1 集合框架的层次结构

Java集合框架是一个统一的架构，用于表示和操作集合。核心接口包括：

Collection接口：代表一组对象的集合
  • List：有序集合，允许重复元素
  • Set：不包含重复元素的集合
  • Queue：用于在处理之前保持元素的集合
Map接口：键值对映射
  • HashMap：基于哈希表的Map实现
  • TreeMap：基于红黑树的Map实现
  • LinkedHashMap：维护插入顺序的HashMap

1.2 与JavaScript的对比

概念 JavaScript Java
数组 Array（动态类型） ArrayList（泛型）
键值对 Object / Map HashMap / TreeMap
唯一值集合 Set HashSet / TreeSet
队列 Array（模拟） LinkedList / ArrayDeque

// Java中的集合

第二部分：HashMap的底层实现

2.1 HashMap的基本结构

HashMap是Java中最常用的集合之一，它基于哈希表实现，提供了O(1)的平均查找时间。

核心组件：
  • 数组（Node[] table）：存储桶（bucket）的数组
  • 链表：解决哈希冲突的链表结构
  • 红黑树：当链表过长时转换为红黑树
  • 哈希函数：将键映射到数组索引

2.2 哈希冲突与链表

当两个不同的键映射到同一个数组索引时，就会发生哈希冲突。Java使用链表来解决这个问题。

数组索引    链表结构
    0       [空]
    1       [Node1] -> [Node2] -> [Node3] -> null
    2       [空]
    3       [Node4] -> null
    4       [空]
    ...
// HashMap的Node节点结构
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;
}

2.3 为什么需要红黑树？

当链表过长时，查找时间会退化为O(n)。为了解决这个问题，Java 8引入了红黑树优化。

链表转红黑树的条件：
  • 链表长度超过8（TREEIFY_THRESHOLD）
  • 数组长度超过64（MIN_TREEIFY_CAPACITY）
重要：红黑树转换是单向的。当红黑树节点数量减少到6（UNTREEIFY_THRESHOLD）时，会退化为链表。

第三部分：红黑树基础

3.1 什么是红黑树？

红黑树是一种自平衡的二叉搜索树，它通过颜色标记和旋转操作来保持树的平衡。

红黑树的五大性质：
  1. 每个节点要么是红色，要么是黑色
  2. 根节点是黑色
  3. 每个叶子节点（NIL）是黑色
  4. 如果一个节点是红色，则它的子节点必须是黑色
  5. 从任一节点到其每个叶子节点的所有路径都包含相同数量的黑色节点

3.2 红黑树的结构

                    [B:13]
                   /      \
              [R:8]        [B:17]
             /    \        /    \
          [B:1]  [B:11]  [R:15]  [B:25]
                /    \         /    \
             [R:6]  [R:14]  [R:22]  [R:27]

            B = 黑色节点, R = 红色节点

3.3 红黑树的优势

数据结构 查找时间 插入时间 删除时间 特点
链表 O(n) O(1) O(1) 简单，但查找慢
二叉搜索树 O(log n) O(log n) O(log n) 可能不平衡
红黑树 O(log n) O(log n) O(log n) 自平衡，性能稳定

4.2 put方法的执行流程

put方法步骤：
  1. 计算键的哈希值
  2. 如果数组为空，进行初始化
  3. 根据哈希值计算数组索引
  4. 如果该位置为空，直接插入新节点
  5. 如果该位置不为空：
    • 如果键相同，覆盖值
    • 如果节点是红黑树节点，插入树中
    • 如果是链表，插入链表尾部
    • 如果链表长度超过8，转换为红黑树
  6. 如果超过负载因子，进行扩容
// 简化的put方法逻辑
public V put(K key, V value) {
    // 1. 计算哈希值
    int hash = hash(key);

    // 2. 计算数组索引
    int index = (n - 1) & hash;

    // 3. 插入节点
    if (table[index] == null) {
        table[index] = new Node<>(hash, key, value, null);
    } else {
        // 处理冲突
        handleCollision(table[index], hash, key, value);
    }

    // 4. 检查是否需要扩容
    if (size > threshold) {
        resize();
    }
}

4.3 为什么负载因子是0.75？

负载因子（Load Factor）是HashMap中用于衡量何时进行扩容的参数。0.75是一个折衷值：
  • 太小：浪费空间，频繁扩容
  • 太大：哈希冲突增多，性能下降

第五部分：实际应用与最佳实践

5.1 如何选择合适的集合？

需求 推荐集合 原因
键值对存储，快速查找 HashMap O(1)平均查找时间
键值对，需要排序 TreeMap 基于红黑树，有序
键值对，保持插入顺序 LinkedHashMap 维护插入顺序
唯一值集合 HashSet 基于HashMap实现
有序唯一值集合 TreeSet 基于TreeMap实现

5.2 使用示例

// 1. HashMap基本使用
Map<String, Integer> map = new HashMap<>();
map.put("apple", 1);
map.put("banana", 2);
int count = map.get("apple");

// 2. 遍历HashMap
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}

// 3. 使用Java 8的forEach
map.forEach((key, value) -> {
    System.out.println(key + ": " + value);
});

5.3 性能优化建议

优化技巧：
  • 预设容量：如果知道大致元素数量，预设HashMap容量
  • 合适的负载因子：根据使用场景调整负载因子
  • 避免频繁扩容：一次性插入大量数据时，预设足够容量
  • 使用不可变键：确保键的hashCode()和equals()方法正确实现
// 预设容量示例
// 预计存储1000个元素，负载因子0.75
// 初始容量 = 1000 / 0.75 ≈ 1333，取2的幂次 = 2048
Map<String, Integer> map = new HashMap<>(2048);
