# 全栈工程师转型学习计划（1-2个月高强度版）

## 学习者背景
- 前端工程师，有JavaScript/TypeScript经验
- 了解Java集合框架（HashMap、HashSet），正在探索红黑树实现
- 无实际Java项目开发经验
- 目标：1-2个月内完成全栈转型

## 学习策略：师徒式指导

### 核心原则
1. **项目驱动**：每个阶段都围绕一个实际项目展开
2. **最小可行产品**：先让代码工作，再深入原理
3. **对比学习**：利用前端经验理解后端概念
4. **即时反馈**：每个课程都有明确的练习和检查点

### 时间分配（每天3-4小时）
- 理论学习：30%
- 动手实践：50%
- 代码审查和调试：20%

## 学习路线（8周计划）

### 第1周：Java基础与Web开发入门
**目标**：能创建Spring Boot项目并实现基本CRUD

#### Day 1-2：Java语言快速入门
- Java语法基础（对比JavaScript）
- 面向对象编程核心概念
- 异常处理
- 练习：实现一个简单的命令行应用

#### Day 3-4：Spring Boot项目搭建
- Spring Boot项目结构
- 依赖注入和控制反转
- 第一个RESTful API
- 练习：创建Hello World API

#### Day 5-7：数据库连接与CRUD
- JPA/Hibernate基础
- 实体类和Repository
- 基本的增删改查操作
- 练习：实现用户管理API

### 第2周：API设计与数据库
**目标**：能设计合理的数据库结构和RESTful API

#### Day 8-9：数据库设计
- 关系型数据库设计原则
- SQL基础（对比MongoDB）
- 索引和性能优化
- 练习：设计电商数据库结构

#### Day 10-11：RESTful API设计
- REST架构原则
- HTTP方法和状态码
- 请求/响应格式设计
- 练习：设计完整的API规范

#### Day 12-14：数据验证与异常处理
- 输入数据验证
- 全局异常处理
- API文档生成（Swagger）
- 练习：完善用户管理API

### 第3周：身份验证与安全
**目标**：能实现基本的身份验证和授权

#### Day 15-16：身份验证基础
- Session vs Token
- JWT身份验证
- Spring Security基础
- 练习：实现登录注册API

#### Day 17-18：授权与权限控制
- 角色和权限管理
- API权限控制
- 安全最佳实践
- 练习：实现基于角色的访问控制

#### Day 19-21：前端集成准备
- CORS跨域处理
- API版本控制

- 练习：创建前端可调用的API

### 第4周：前端集成与全栈开发
**目标**：能完成前后端联调

- 状态管理

#### Day 24-25：前后端联调
- 前端调用后端API
- 数据流和状态同步
- 错误处理和加载状态

#### Day 26-28：功能完善
- 分页和搜索
- 文件上传
- 实时通知（WebSocket）
- 练习：完善核心功能

### 第5周：测试与质量保证
**目标**：能编写测试并保证代码质量

#### Day 29-30：单元测试
- JUnit 5基础
- Service层测试
- Repository层测试

#### Day 31-32：集成测试
- API测试
- 数据库测试
- 测试覆盖率
- 练习：编写完整的集成测试

#### Day 33-35：代码质量
- 代码规范和最佳实践
- 代码审查要点
- 性能优化基础
- 练习：重构和优化现有代码

### 第6周：部署与DevOps
**目标**：能将应用部署到云服务器

#### Day 36-37：Docker容器化
- Docker基础概念
- Dockerfile编写
- Docker Compose
- 练习：容器化Spring Boot应用

#### Day 38-39：云服务部署
- 云服务基础（AWS/阿里云）
- 数据库部署
- 应用部署
- 练习：部署到云服务器

#### Day 40-42：CI/CD流水线
- GitHub Actions/GitLab CI
- 自动化测试
- 自动化部署
- 练习：创建完整的CI/CD流水线

### 第7-8周：项目实战
**目标**：独立开发并部署完整的Web应用

#### 项目选择（任选其一）
1. **博客系统**：用户管理、文章发布、评论系统
2. **电商后台**：商品管理、订单处理、用户管理
3. **任务管理**：项目管理、任务分配、进度跟踪

#### 开发流程
1. 需求分析和设计（2天）
2. 数据库设计和API设计（2天）
3. 后端开发（4天）
4. 前端开发（4天）
5. 测试和调试（2天）
6. 部署上线（2天）

## 常见问题与解决方案

### Q1：如何平衡学习深度和速度？
A：先让代码工作，再深入原理。每个知识点先掌握20%的核心内容，解决80%的问题。

### Q2：遇到问题怎么办？
A：1. 查看官方文档 2. 搜索Stack Overflow 3. 使用AI辅助 4. 记录问题并总结

### Q3：如何保持学习动力？
A：设定小目标，及时奖励自己。每周回顾进步，保持正向反馈。

### Q4：前端经验如何利用？
A：对比学习JavaScript和Java的差异，利用已有的编程思维理解新概念。
