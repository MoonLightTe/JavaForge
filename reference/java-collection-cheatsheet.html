📋 核心接口速查

Collection接口

接口 特点 常用实现
List 有序，可重复 ArrayList, LinkedList

Map接口

实现 特点 时间复杂度
HashMap 基于哈希表 O(1) 平均
TreeMap 基于红黑树，有序 O(log n)
LinkedHashMap 维护插入顺序 O(1) 平均

🔧 HashMap核心参数

// 关键参数
DEFAULT_INITIAL_CAPACITY = 16
DEFAULT_LOAD_FACTOR = 0.75f

UNTREEIFY_THRESHOLD = 6

关键点：
  • 链表转红黑树：链表长度 > 8 且数组长度 > 64
  • 红黑树退化为链表：节点数 < 6
  • 扩容：元素数量 > 容量 × 负载因子

🌳 红黑树性质

                [B:13]
               /      \
          [R:8]        [B:17]
         /    \        /    \
      [B:1]  [B:11]  [R:15]  [B:25]

            B = 黑色, R = 红色
五大性质：
  1. 节点颜色：红或黑
  2. 根节点：黑色
  3. 叶子节点(NIL)：黑色
  4. 红色节点的子节点：必须黑色
  5. 黑色节点数：从任一节点到叶子路径上的黑色节点数相同
优势：红黑树通过这五大性质保证树的平衡，查找、插入、删除时间复杂度均为O(log n)。

⚡ 性能对比

数据结构 查找 插入 删除 适用场景
链表 O(n) O(1) O(1) 少量数据，频繁插入
红黑树 O(log n) O(log n) O(log n) 大量数据，需要有序
哈希表 O(1) 平均 O(1) 平均 O(1) 平均 快速查找，无序

🎯 集合选择指南

选择Map

  • 快速查找，无序 → HashMap
  • 需要排序 → TreeMap
  • 保持插入顺序 → LinkedHashMap
  • 线程安全 → ConcurrentHashMap

选择Collection

  • 有序列表 → ArrayList
  • 频繁插入/删除 → LinkedList
  • 唯一值集合 → HashSet
  • 有序唯一值 → TreeSet

📝 常用代码片段

HashMap基本操作

// 创建
Map<String, Integer> map = new HashMap<>();

// 添加
map.put("key", 1);

// 获取
int value = map.get("key");

// 检查
boolean exists = map.containsKey("key");

// 遍历
map.forEach((k, v) -> System.out.println(k + ": " + v));

预设容量（性能优化）

// 预计存储1000个元素
// 容量 = 1000 / 0.75 ≈ 1333
// 取2的幂次 = 2048
Map<String, Integer> map = new HashMap<>(2048);

⚠️ 常见陷阱

1. 键对象必须正确实现hashCode()和equals()
否则会导致查找失败或重复元素。
2. HashMap不是线程安全的
多线程环境使用ConcurrentHashMap或Collections.synchronizedMap()。
3. 避免在遍历时修改Map
会抛出ConcurrentModificationException，使用Iterator的remove()方法。
4. TreeMap要求键实现Comparable
否则需要在构造函数中提供Comparator。
